# Data Model (Phase 1): Content Architecture

This project is a static site, so the “data model” is a content model: doc types, required metadata (frontmatter), and relationships used for navigation.

## Entities

### Module

- **Represents**: A top-level learning unit (Module 1–4).
- **Implementation**: Sidebar category (`sidebars.ts`) containing ordered chapter docs.
- **Fields (conceptual)**:
  - `moduleId` (string, e.g. `module-01`)
  - `title` (string)
  - `description` (string)
  - `chapters` (ordered list of Chapter IDs)

### Chapter

- **Represents**: A single lesson page.
- **Implementation**: A Markdown/MDX doc under `docs/` (commonly `docs/chapters/`).
- **Recommended frontmatter fields**:
  - `id` (string, unique)
  - `title` (string)
  - `description` (string, optional)
  - `sidebar_position` (number, optional; if using autogenerated ordering)
  - `tags` (string[], optional)
  - `module` (string, e.g. `module-01`)
  - `prerequisites` (string[], optional)
  - `learningObjectives` (string[], optional)
  - `estimatedTimeMinutes` (number, optional)

### Lab

- **Represents**: A hands-on exercise tied to a chapter/module.
- **Implementation**: Either:
  - a dedicated doc under `docs/labs/`, or
  - a section within a chapter (if short).
- **Recommended frontmatter fields (when standalone)**:
  - `id`, `title`, `module`, `chapterId` (string)
  - `difficulty` (string enum: `beginner|intermediate|advanced`)
  - `estimatedTimeMinutes` (number)
  - `requirements` (string[], e.g. software/hardware prerequisites)

### Quiz

- **Represents**: A self-check assessment tied to a chapter/module.
- **Implementation**: Either:
  - a dedicated doc under `docs/quizzes/`, or
  - an embedded section in a chapter using collapsible answers.
- **Recommended frontmatter fields (when standalone)**:
  - `id`, `title`, `module`, `chapterId` (string)
  - `questionCount` (number)
  - `estimatedTimeMinutes` (number, optional)

### Asset

- **Represents**: Images/diagrams used by docs.
- **Implementation**: Files under `static/img/` referenced as `/img/<file>`.
- **Fields (conceptual)**:
  - `path` (string)
  - `alt` (string)
  - `source` (string, optional attribution)

## Relationships

- **Module → Chapters**: 1-to-many (ordered in `sidebars.ts`).
- **Chapter → Labs/Quizzes**: 0-to-many; may be embedded or referenced via links.
- **Docs → Assets**: many-to-many via Markdown references.

## Validation Rules

- All doc `id` values must be unique across the site.
- Any internal link must resolve at build time (`onBrokenLinks: "throw"`).
- Any referenced image under `/img/...` must exist under `static/img/`.
